Chapter 01

Global Hotkey & Window Management

The entry point for any launcher is the global hotkey. Crest uses the global-hotkey Rust crate to listen for system-wide key events even when the app isn't focused.

  • Mechanism: Rust backend registers a persistent listener for Super+Space (configurable).
  • Logic: When triggered, the backend sends a window-toggle signal. If the window is hidden, it shows and focuses; if visible, it hides.
  • Optimization: We use an async event loop to ensure zero-latency response between the keypress and the UI appearance.
Chapter 02

Unified Search Engine

The "brain" of Crest processes user queries in real-time. It's designed for sub-millisecond ranking across multiple sources.

  • Engine: Powered by the nucleo crate for high-performance fuzzy matching.
  • Mechanism: Queries are sent from the frontend on every keystroke and processed by the Rust backend in parallel.
  • Prioritization: Results are weighted based on source type (Apps > Commands > Files) and usage frequency.
Chapter 03

Data Indexers

To keep searches instantaneous, Crest doesn't scan the disk live. Instead, it maintains a persistent local index.

  • Sources: Indexers scan .desktop files, system binaries, and user-defined directories.
  • Storage: Metadata is stored in a local SQLite database for ultra-fast retrieval.
  • Refresh: Background workers periodically check for changes to ensure the index stays current without eating CPU.
Chapter 04

Plugin System

Extensibility is at the heart of Crest. Our plugin system allows anyone to add new functionality with simple scripts.

  • Architecture: Manifest-driven (manifest.json). Only explicitly registered plugins are loaded.
  • Security: Plugins run as isolated processes. They communicate via JSON stdout, keeping the core app safe.
  • Standardization: Any language can be used (Node, Python, Bash) as long as it outputs the Crest Result Schema.
Chapter 05

Tauri IPC Bridge

The communication channel between our Rust "muscles" and React "skin."

  • Mechanism: Uses Tauri's optimized invoke and emit patterns.
  • Data Flow: Keystrokes flow down to Rust; ranked results flow back up to the TypeScript layer.
  • Latency: By keeping the IPC payloads small and structured, we maintain a consistent 60+ FPS search experience.
Chapter 06

Frontend State Management

The frontend is responsible for the "Visual Revolution"—rendering results with high-fidelity aesthetics and smooth motion.

  • State: Managed via Zustand for lightweight, reactive updates without the overhead of Redux.
  • Motion: Powered by Framer Motion for glassy transitions and spring-based selection highlights.
  • Accessibility: Full keyboard-driven navigation (arrows, enter, shortcuts) is handled via dedicated hooks.
